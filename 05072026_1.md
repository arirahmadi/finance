# Dokumentasi Pengembangan Sistem Keuangan (Finance System)
**Waktu Rilis:** 05 Juli 2026, 17:30 WIB
**Nama Berkas:** 05072026_1.md

Dokumentasi ini mencatat daftar tugas (tasks) pengembangan tambahan yang dikerjakan pada tanggal 05 Juli 2026 secara komprehensif, mencakup perbaikan visual, perbaikan bug edit, hapus massal, hingga sistem manajemen user roles dan settlement advance.

---

## 1. Daftar Tugas (Task List) & Status Pengerjaan

### 1.1 Perbaikan Kontras Highlight Hover Sidebar
* **Status:** `Selesai`
* **Deskripsi:** Perbaikan warna latar hover pada sidebar agar teks menu (seperti Transaksi) tetap kontras dan terbaca dengan tajam (tidak tertutup warna latar hover).
* **Solusi:** Menambahkan `z-index: 1` pada item menu navigasi dan `z-index: -1` pada pseudo-element hover latar di file `public/css/dashboard.css`.

### 1.2 Perbaikan Fungsi Edit Transaksi
* **Status:** `Selesai`
* **Deskripsi:** Memperbaiki bug di mana modal edit tidak muncul akibat kesalahan parsing parameter kutip deskripsi transaksi di Blade.
* **Solusi:** Memanfaatkan dataset JSON (`data-tx="{{ json_encode($tx) }}"`) dan mem-parsing-nya secara dinamis via JavaScript saat tombol edit diklik.

### 1.3 Fitur Delete Bulk (Hapus Massal)
* **Status:** `Selesai`
* **Deskripsi:** Penambahan fitur checklist pilihan transaksi pada tabel utama untuk menghapus beberapa transaksi sekaligus secara aman.
* **Solusi:** Endpoint backend `DELETE /transactions/bulk` dengan validasi kepemilikan owner dan penghapusan fisik file bukti transaksi dari disk storage.

### 1.4 Sistem Hak Akses Dinamis User (User Roles)
* **Status:** `Selesai`
* **Deskripsi:** Menu pengaturan hak akses di mana Owner dapat menambahkan pengguna staff baru dan menentukan izin akses secara dinamis (Lihat Transaksi, Tambah Transaksi, Edit Transaksi, Hapus Transaksi, dan COA).
* **Solusi:**
  * Penambahan kolom `permissions` (JSON array) ke tabel `users`.
  * Metode helper `$user->hasPermission($permission)` pada model User.
  * Antarmuka tab **User & Roles** (hanya terlihat oleh Owner) untuk menambah user, mengubah permissions, dan reset password.
  * Proteksi route dan tombol edit/delete transaksi di backend & frontend sesuai permissions pengguna.

### 1.5 Menu Settlement & Advance (Uang Muka Pembelian)
* **Status:** `Selesai`
* **Deskripsi:** Sistem pencatatan uang muka keluar (Advance), perhitungan otomatis outstanding debt (hutang karyawan), dan form pelaporan bon fisik (Settlement) dengan penyesuaian selisih kas perusahaan.
* **Solusi:**
  * Penambahan kolom `is_advance`, `advance_status`, `settled_at`, dan `settlement_amount` pada tabel `transactions`.
  * Pendaftaran akun aset lancar baru **`1202 - Uang Muka Pembelian`** ke Chart of Accounts (COA).
  * Antarmuka tab **Settlement** yang menampilkan total outstanding advance (hutang karyawan yang belum lunas) dan daftar advance.
  * Form "Buat Uang Muka (Advance)" yang mendebit akun `1202` dan mengkredit akun Kas/Bank.
  * Form "Laporkan Bon (Settlement)" dengan kewajiban upload berkas bon asli dan penentuan akun beban riil. Sistem secara otomatis menghitung selisih kas (kelebihan uang muka didebit kembali ke kas, kekurangan uang muka dikredit keluar dari kas) untuk menjamin pembukuan double-entry tetap seimbang.
