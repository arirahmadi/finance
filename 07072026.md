## Sesi Pagi (09:00 – 12:00 WIB)

### 1.1 Otorisasi Granular Menu Settlement (Role-Based Access Control)
* **Deskripsi:** Memisahkan hak akses menu Settlement sepenuhnya dari izin transaksi umum, sehingga Owner dapat mengontrol akses staff secara granular.
* **Perubahan Database & Seeder:**
  * Memperbarui [DatabaseSeeder.php](file:///Users/arirahmadi/Documents/system/finance-sys/database/seeders/DatabaseSeeder.php) untuk mendaftarkan default permission baru ke staff `admin@finance.com`.
* **Hak Akses Baru yang Diterapkan:**
  | Izin (Permission) | Teks Tampilan | Fungsionalitas |
  |---|---|---|
  | `view_settlements` | Lihat Menu Settlement | Menampilkan/menyembunyikan link menu "Settlement" di sidebar navigasi. |
  | `create_settlements` | Tambah Settlement (Uang Muka) | Mengizinkan pembuatan transaksi pengeluaran Uang Muka baru. |
  | `process_settlements` | Proses Settlement (Settle) | Mengizinkan pelaporan kuitansi/bon fisik untuk uang muka yang menggantung. |
  | `delete_settlements` | Hapus Settlement (Delete) | Mengizinkan penghapusan data uang muka / pertanggungjawaban baik satuan maupun massal. |
* **Antarmuka Pengguna (UI):**
  * Menambahkan checkbox konfigurasi izin baru tersebut pada modal **Tambah User** dan **Edit User** di tab *User & Roles* agar Owner dapat mencentang/menghapus hak akses secara langsung.

### 1.2 Fitur Hapus (Single Delete) & Bulk Delete Settlement
* **Deskripsi:** Memberikan fasilitas bagi Owner (atau staff berizin) untuk menghapus satu atau banyak data settlement sekaligus secara bersih dan aman.
* **Implementasi Backend:**
  * Endpoint baru: `DELETE /settlements/{id}` untuk menghapus satu settlement.
  * Endpoint baru: `DELETE /settlements/bulk` untuk menghapus beberapa settlement yang dipilih.
  * Logika controller menghapus catatan transaksi, jurnal entri terkait (debit/kredit), serta berkas kuitansi fisik dari server lokal (`public/receipts/`).
* **Implementasi Frontend:**
  * Kolom checkbox (select-all di header dan centang individu di setiap baris data) ditambahkan pada tabel Daftar Uang Muka.
  * Tombol **"Hapus Terpilih"** berwarna merah muncul dinamis di atas tabel ketika terdapat data yang dicentang.
  * Form hidden ditambahkan untuk mengirimkan data array ID yang terpilih ke backend.

### 2.2 Fitur Ubah Settlement (Edit)
* **Deskripsi:** Memungkinkan pengguna berizin untuk mengoreksi kesalahan entri data uang muka maupun data laporan bon fisik tanpa perlu menghapus dan mengulang pengisian dari awal.
* **Konfigurasi Hak Akses:**
  * Menambahkan permission `edit_settlements` ("Ubah Settlement (Edit)") ke dalam pilihan perizinan user.
* **Fungsionalitas Modal Ubah (Edit Modal):**
  * Tombol Edit (ikon pensil 📝) disediakan di kolom Aksi.
  * Form modal secara dinamis mendeteksi status settlement:
    * **Status "Belum Dilaporkan":** Hanya menampilkan input data dasar uang muka (Tanggal, Nama Penerima, Nominal Uang Muka, Sumber Kas, Keterangan).
    * **Status "Selesai (Settled)":** Menampilkan input data dasar PLUS data laporan kuitansi (Nominal Riil Bon, Akun Kategori Beban, dan kolom unggah file baru untuk menggantikan file bon lama).
    * **Kalkulator Selisih:** Dilengkapi kalkulator selisih real-time yang memperbarui informasi kurang/lebih bayar saat nominal diubah.
* **Keamanan Buku Besar (Double-Entry Balance):**
  * Backend membungkus pembaruan data di dalam `DB::transaction`. Sistem menghapus entri jurnal lama milik transaksi tersebut, lalu menulis ulang entri jurnal baru dengan nominal/akun kas baru, memastikan kas utama dan pencatatan debit/kredit tetap balance sempurna.
