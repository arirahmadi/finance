## Sesi Sore (13:00 – 17:57 WIB)

### 4.1 Menu Settlement & Advance (Sistem Uang Muka Pembelian)
**13:20 WIB** | Status: `Selesai`
* **Deskripsi:** Sistem pencatatan uang muka keluar (Advance), perhitungan outstanding debt (hutang yang belum dilaporkan bon), dan form pelaporan bon fisik (Settlement).
* **Logika Bisnis:**
  * Jika ada advance pembelian server (atau pembelian apapun) yang keluar dari kas, dicatat sebagai **Advance** dengan status `open`.
  * Karyawan **wajib** melaporkan pembelian dengan bukti bon fisik (foto/PDF).
  * Selama belum ada laporan bon, advance tersebut **terhitung sebagai hutang karyawan** (outstanding).
  * Setelah bon diunggah dan dilaporkan, status advance berubah menjadi `settled` dan jurnal penyesuaian dibuat otomatis.
* **Implementasi Database:**
  * Kolom baru pada tabel `transactions`: `is_advance` (boolean), `advance_status` (`open`/`settled`), `settled_at` (timestamp), `settlement_amount` (decimal).
  * Akun COA baru: **`1202 - Uang Muka Pembelian`** (aset lancar).
* **Mekanisme Jurnal Buku Ganda:**
  * **Saat Advance Dibuat:** `Debit 1202 - Uang Muka Pembelian` / `Kredit 1101 - Kas` (atau bank).
  * **Saat Settlement (bon dilaporkan):**
    * `Debit [Akun Beban]` (sesuai bon nyata, misal: Biaya Server) / `Kredit 1202 - Uang Muka Pembelian`.
    * Jika terdapat **selisih lebih** (uang muka > biaya aktual): Sisa uang dikembalikan → `Debit 1101 - Kas`.
    * Jika terdapat **selisih kurang** (uang muka < biaya aktual): Perusahaan nombok → `Kredit 1101 - Kas`.
* **Tab Settlement UI:**
  * Ringkasan total **Outstanding Advance** (outstanding hutang karyawan).
  * Ringkasan total **Advance Lunas**.
  * Tabel daftar advance dengan status, tanggal, nominal, dan aksi.
  * Modal **Buat Uang Muka (Advance)** dengan dropdown sumber kas.
  * Modal **Laporkan Bon (Settlement)** dengan upload wajib bukti bon, pilihan akun beban, dan nominal aktual.

### 4.3 Perbaikan Bug: User `admin@finance.com` Tidak Muncul di Tabel User & Roles
**15:00 WIB** | Status: `Selesai`
* **Deskripsi:** Pengguna `admin@finance.com` sudah ada di database tetapi tidak tampil di tabel **User & Roles** pada tab settings.
* **Analisis Root Cause:**
  * Rute utama `/` (dashboard) hanya memanggil method `index()` di controller, tetapi method tersebut tidak meload data `$users` ke view.
  * Method `indexUsers()` memiliki query load users sendiri, tetapi rute `/settings/users` meredirect ke method tersebut secara terpisah dan tidak terintegrasi ke SPA.
  * Saat user mengklik tab "User & Roles" di JavaScript SPA, data `$users` sudah tidak tersedia karena view-nya di-render oleh rute `/` yang tidak memiliki `$users`.
* **Solusi:**
  * Meng-update method `index()` di [WebController.php](file:///Users/arirahmadi/Documents/system/finance-sys/app/Http/Controllers/Web/WebController.php) untuk selalu meload `$users` (jika user adalah owner), data advances, dan settlement summary secara default.
  * Meng-update method `indexUsers()` dan `indexSettlements()` untuk cukup me-redirect ke rute utama `/` dengan query parameter `?activeTab=users` / `?activeTab=settlements`.
  * Meng-update listener `DOMContentLoaded` di JavaScript agar membaca URL query parameter `activeTab` sehingga tab yang dimaksud langsung aktif setelah redirect.
